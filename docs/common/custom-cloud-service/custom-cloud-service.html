What is a custom cloud service

A custom cloud service combines many cloud functions like the cloud services exposed by ZetaPush. The only difference is that it is you that create the cloud functions. Generally, we put the business logic of your application in the custom cloud services.

Example

Your use-case is a basic real-time chat between characters of Avengers (See tutorial). You create the chat with the cloud services provided by ZetaPush and you want to add a new feature : Each Avenger has many skills/attacks and they can randomly use one of them on the chat.

To add this business logic, you can create a custom cloud service named AvengerService and into this, you add a cloud function named launchRandomAttack.

Dependencies

If you use existing cloud services provided by ZetaPush, you need to import them from @zetapush/platform. So you need to install this dependency with :

$ npm install --save @zetapush/platform

Register / expose my custom cloud service

When you create a custom cloud service you need to register it. Then, if you want to use it on the client side, you need to expose it.

Example

It is considered that we have 3 custom cloud services :

  • Services exposed for the client side

    • MyCustomCloudService1

    • MyCustomCloudService2

  • Private service not exposed to the client (pur business logic)

    • MyPrivateCustomCloudService

You need to register custom cloud services in the file indicated by the property main in the package.json

export default {

   myCustomCloudService2: MyCustomCloudService2
  },
  internal: { (2)
   myPrivateCustomCloudService: MyPrivateCustomCloudService
  }

const { Stack } = require('@zetapush/platform')

class DateCustomService {
    constructor() {}

    formatDate(date) {
        return date.toLocalDateString("fr-FR");
    }
}

class SimpleStorageWithDate {
    constructor(stack, dateCustomService) {
        this.stack = stack;
        this.dateCustomService = dateCustomService;
    }

    async push(item) { (1)
        const output = await this.stack.push({ (2)
            stack: 'test',
            data: {
                value: item,
                date: this.dateService.formateDate(new Date()); (3)
            }
        });
        return const;
    }

    async list() { (1)
        const output = await this.stack.list({ stack: 'demo'});
        return output;
    }
}

export default { (4)
  exposed: {
   simpleStorageWithDate: SimpleStorageWithDate,
  },
  internal: {
   dateCustomService: DateCustomService
  }
}
1 Async/Await context
2 Use the existing cloud services
3 Use an other custom cloud service
